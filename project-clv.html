Customer Lifetime Value Analysis in E-Commerce

The Problem

A fast-growing e-commerce business had been relying on third-party tools to understand user behaviour. As the company scaled, those tools were insufficient. The core business question was concrete: was the $2 per customer marketing acquisition cost actually justified? Without a rigorous CLV framework, there was no way to answer that — only instinct.

The Approach

Working with a parsed events table (raw_events) containing front-end user actions from November 2020 to January 2021, I filtered purchase events and used user_pseudo_id as the unique customer identifier. I built a BigQuery SQL framework using CTEs and window functions to calculate a sequential chain of metrics:

Average Order Value (AOV) → Purchase Frequency (PF) → Customer Value → Customer Lifetime Value (CLV), where CLV was derived as Customer Value multiplied by an assumed 3-year customer lifespan — a parameter made explicit and documented rather than buried in the code.

The SQL used layered CTEs so each step was auditable and the assumptions were visible to anyone reviewing the model.

Key Findings

The average CLV was $2,950.43 — confirming the $2 acquisition cost was not just justified but represented an extraordinary return on investment by a factor of over 1,400.

Purchase frequency analysis revealed which customer segments warranted higher re-engagement spend — separating high-value loyalists from one-time buyers with meaningfully different long-term value profiles.

AOV and frequency data together enabled value-based segmentation that third-party summaries could not provide — giving the marketing team a defensible framework for budget allocation across acquisition channels.

Impact

The analysis gave the marketing team a quantified, reproducible framework for acquisition spend decisions. Knowing that CLV dwarfs acquisition cost at scale justifies sustained investment in customer acquisition — and creates a documented baseline for testing whether that ratio improves or erodes over time.

The CLV calculation was also set up as a reusable SQL model so the business could run it against updated data each quarter without rebuilding the logic.

What I Learned

"The most important step in CLV analysis is agreeing on the definition of 'lifetime' before writing any SQL. Using different assumptions changes the output dramatically. For this project I used 3 years, but I would always make that a documented parameter rather than a hardcoded value in future work — so the business can own and update the assumption rather than inheriting mine."
